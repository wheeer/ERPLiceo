## 🎯 ESPECIFICACIÓN DEL LOGIN (CRÍTICA)

### CONTEXTO DE DISEÑO
- 🌙 **Tema**: Oscuro, elegante, maduro, sobriedad (no gamificado)
- 🏔️ **Fondo**: Montañas + Cielo nocturno + Nebulosas (Aurora)
- 💳 **Tarjeta**: Semi-transparente, textura mate, flotante, profundidad
- ✨ **Efecto**: Iluminación detrás (cuando focus), movimiento sutil

### LAYOUT
```
┌─────────────────────────────────────┐
│         AURORA (Fondo)              │
│     (Montañas + Nebulosas)          │
│                                     │
│                    ┌──────────────┐ │
│                    │  FORMULARIO  │ │
│                    │  (Tarjeta)   │ │
│                    │              │ │
│                    │  RUT         │ │
│                    │  Contraseña  │ │
│                    │  [Ingresar]  │ │
│                    └──────────────┘ │
│                                     │
│  [Logo] [Nombre Colegio]           │
└─────────────────────────────────────┘
```

### INTERACTIVIDAD REQUERIDA
- ✅ Focus en inputs → movimiento + iluminación (sutil)
- ✅ Pantalla de carga (spinner + mensajes progresivos)
- ✅ Errores con iconos + textos claros + soluciones
- ✅ Validación visual RUT (tiempo real)
- ✅ Estados: default, focus, hover, error, loading

### RESPONSIVE
- 📱 **Mobile (375px)**: Tarjeta centrada, simplificada
- 📲 **Tablet (768px)**: Ajuste proporcional
- 🖥️ **Desktop (1920px)**: Layout original propuesto
- ⚡ **Feedback visual**: Adaptado, NO exagerado

### CSS
- 📄 **Archivo**: `src/app/features/auth/login/login.css` (SEPARADO)
- 🎨 **Variables**: Tema oscuro (colores, sombras, transiciones)
- 🔀 **No contamina**: Otros módulos mantienen su diseño
- ♻️ **Reutilizable**: Variables globales en `src/styles.css`

## 🎨 PRINCIPIOS DE DISEÑO (INNEGOCIABLES)

### HEURÍSTICAS DE NIELSEN
1. ✅ **Visibilidad del sistema**: Estados claros (loading, error, success)
2. ✅ **Control del usuario**: Libertad para navegar
3. ✅ **Consistencia**: Mismo estilo en todo
4. ✅ **Prevención de errores**: Validaciones visuales
5. ✅ **Diseño estético minimalista**: Nada decorativo sin función

### VALIDACIÓN ANTES DE IMPLEMENTAR
- ❌ NO diseñar "porque se ve bonito"
- ❌ NO interfaces recargadas
- ❌ NO elementos sin feedback
- ✅ TODO elemento tiene propósito claro
